# Penetration Testing Skill

Defensive security assessments and authorized penetration testing methodology.

## Allowed Use

- Authorized environments, labs, and owned infrastructure
- Bug bounty programs with explicit scope
- Internal security audits and compliance checks

## Disallowed Use

- Testing targets without explicit written permission
- Data exfiltration, credential reuse, or harmful actions
- Chain support for malicious recon or exploitation

## Triggers

- penetration testing
- vulnerability assessment
- security audit
- bug bounty
- red team
- vulnerability research
- security scanning

## Workflow

1. Define scope, targets, and authorization proof
2. Enumerate assets and validate testing boundaries
3. Map attack surface with non-destructive checks
4. Identify vulnerabilities with impact and evidence
5. Produce remediation guidance and severity ranking

## Inputs

- Scope and target list
- Authorization confirmation
- Rules of engagement
- Environment details and credentials for authenticated testing if approved

## Outputs

- Attack surface summary
- Vulnerability findings with severity and impact
- Evidence and reproduction steps
- Remediation guidance and prioritized fixes

## Safety

- Scope enforcement before every action
- No destructive payloads or unauthorized access
