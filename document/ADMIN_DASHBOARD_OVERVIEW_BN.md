# সুপ্রিমএআই: এডমিন ড্যাশবোর্ড ফিচার ওভারভিউ (Admin Dashboard Features Overview)

এই ডকুমেন্টটি সুপ্রিমএআই প্রকল্পের এডমিন ড্যাশবোর্ডের বর্তমান এবং পরিকল্পিত ফিচারগুলোর একটি বিস্তারিত চিত্র তুলে ধরে। এটি `Contract-Driven UI` আর্কিটেকচারের মাধ্যমে পরিচালিত হয়, যা ব্যাকএন্ড (Java) থেকে React এবং Flutter ইন্টারফেস নিয়ন্ত্রণ করে। এর ফলে ফ্রন্টএন্ডে কোড পরিবর্তনের প্রয়োজনীয়তা কমে আসে।

---

## ১. এডমিন ড্যাশবোর্ড আর্কিটেকচার

*   **আর্কিটেকচার:** `Contract-Driven UI` (ব্যাকএন্ড থেকে UI নিয়ন্ত্রণ)।
*   **ফ্রন্টএন্ড টেকনোলজি:** React এবং Flutter।
*   **লক্ষ্য:** ফ্রন্টএন্ডে কোড পরিবর্তনের প্রয়োজনীয়তা কমানো এবং ডাইনামিক কনফিগারেশন সক্ষমতা বাড়ানো।

### বর্তমান এবং পরিকল্পিত ড্যাশবোর্ড ফিচারসমূহ:

*   **সিস্টেম কনফিগারেশন ম্যানেজমেন্ট:**
    *   `SystemWorkRuleService` এর মাধ্যমে সমস্ত সিস্টেম রুলস (যেমন `VOTING_CONSENSUS_THRESHOLD`, `FIRECRAWL_API_KEY`, `FABRIC_PATTERNS_PATH`) ডাইনামিকভাবে কনফিগার করা যায়।
    *   **স্ট্যাটাস:** **সক্রিয়** (এডমিন `SystemWorkRule` গুলোকে Firestore-এ আপডেট করতে পারেন)।
*   **এআই প্রোভাইডার ম্যানেজমেন্ট:**
    *   প্রতিটি এআই প্রোভাইডারের `canParticipateInVoting` ফ্ল্যাগ এবং `weight` ড্যাশবোর্ড থেকে নিয়ন্ত্রণ করা যায়।
    *   প্রোভাইডারদের স্বাস্থ্য (health) এবং কোয়ারেন্টাইন স্ট্যাটাস মনিটর করা যায়।
    *   **স্ট্যাটাস:** **সক্রিয়** (`MultiAIVotingService` এর মাধ্যমে ডাইনামিক ভোটিং কন্ট্রোল করা যায়)।
*   **ইউজার এবং পারমিশন ম্যানেজমেন্ট:**
    *   ইউজারদের রোল (যেমন ADMIN) অ্যাসাইন করা এবং API Key Revocation (`um-revoke-01`) এর মতো কাজগুলো ড্যাশবোর্ড থেকে করা যায়।
    *   **স্ট্যাটাস:** **সক্রিয়** (Firestore `users` এবং `user_api_keys` কালেকশন ব্যবহার করে)।
*   **মনিটরিং এবং হেলথ চেক:**
    *   `DEPLOYMENT_STATUS_AND_USAGE.md` অনুযায়ী, `collecthealthmetrics` সার্ভিসের মাধ্যমে খরচ ট্র্যাক করা এবং `actuator/health` এন্ডপয়েন্টের মাধ্যমে সিস্টেমের স্বাস্থ্য দেখা যায়।
    *   **স্ট্যাটাস:** **সক্রিয়** (তবে Sentry এবং PostHog ইন্টিগ্রেশন পরিকল্পনাধীন)।
*   **ওয়ার্কফ্লো অটোমেশন (n8n):**
    *   `n8n` ওয়ার্কফ্লো ড্যাশবোর্ড থেকে কনফিগার এবং মনিটর করা যায়, যা থার্ড-পার্টি API ইন্টিগ্রেশন এবং জটিল অটোমেশন চেইন তৈরি করে।
    *   **স্ট্যাটাস:** **সক্রিয়** (Cloud Run-এ ডিপ্লয়ড)।
*   **জ্ঞানভাণ্ডার ব্যবস্থাপনা (Knowledge Base Management):**
    *   `SYSTEM_LEARNING` এবং `CORE_KNOWLEDGE` এন্ট্রিগুলো ড্যাশবোর্ড থেকে পর্যালোচনা, আপডেট এবং নতুন এন্ট্রি যোগ করার ক্ষমতা।
    *   **স্ট্যাটাস:** **পরিকল্পনাধীন** (বর্তমানে `KnowledgeSeedDataProvider` এর মাধ্যমে সিড করা হয়)।
*   **এআই মডেল ট্রায়াজ এবং রাউটিং:**
    *   `TRIAGE_MODEL_ID` এর মতো রুলস ড্যাশবোর্ড থেকে সেট করা যায়, যা এআই মডেল রাউটিং স্ট্র্যাটেজি নিয়ন্ত্রণ করে।
    *   **স্ট্যাটাস:** **সক্রিয়** (`SystemWorkRuleService` এর মাধ্যমে)।
*   **ব্রাউজার অটোমেশন এবং স্ক্র্যাপিং:**
    *   `Firecrawl.dev` এর API Key এবং `Playwright.dev` এর কনফিগারেশন ড্যাশবোর্ড থেকে নিয়ন্ত্রণ করা যায়।
    *   **স্ট্যাটাস:** **সক্রিয়** (`FirecrawlService` ইমপ্লিমেন্ট করা হয়েছে)।
*   **মেমরি এবং লার্নিং লেয়ার:**
    *   `Claude-mem` বা `Mem0` এর API Key এবং MCP সার্ভার URL ড্যাশবোর্ড থেকে কনফিগার করা যায়।
    *   **স্ট্যাটাস:** **সক্রিয়** (`ClaudeMemoryService` ইমপ্লিমেন্ট করা হয়েছে)।

### ভবিষ্যৎ ড্যাশবোর্ড ফিচারসমূহ:
*   **Vector Search Integration:** ড্যাশবোর্ড থেকে ভেক্টর ডাটাবেস (Turbovec, Pinecone) কনফিগারেশন এবং সার্চ রেজাল্ট মনিটরিং।
*   **Automated Marketing Pipeline:** Higgsfield API ব্যবহার করে ডেমো ভিডিও জেনারেশন এবং মার্কেটিং ক্যাম্পেইন ম্যানেজমেন্ট।
*   **Zero-Code Scaling:** ড্যাশবোর্ড কন্ট্রাক্ট আরও উন্নত করে নতুন ফিচার লঞ্চের সময় কোডিং কমানো।
*   **Sentry/PostHog ইন্টিগ্রেশন:** এরর ট্র্যাকিং এবং ইউজার অ্যানালিটিক্স ড্যাশবোর্ডে সরাসরি দেখা।
