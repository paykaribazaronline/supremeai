# সুপ্রিম এআই (SupremeAI) - কমান্ড গাইড

এই ডকুমেন্টে SupremeAI প্রজেক্ট ডেভেলপমেন্ট, ডেপ্লয়মেন্ট এবং মেইনটেইন করার জন্য প্রয়োজনীয় সকল কমান্ডের একটি তালিকা দেওয়া হলো। সাথে প্রতিটি কমান্ড কেন ব্যবহার করা হয়, তার ব্যাখ্যাও দেওয়া হয়েছে।

## ১. ডকার (Docker) কমান্ডসমূহ
ডকার ব্যবহার করে পুরো প্রজেক্ট একসাথে রান করার জন্য এই কমান্ডগুলো প্রয়োজন।

| কমান্ড | কারণ/ব্যবহার |
| :--- | :--- |
| `docker compose up --build -d` | পুরো প্রজেক্ট (ব্যাকএন্ড, ফ্রন্টএন্ড, ডেটাবেস ইত্যাদি) বিল্ড করে ব্যাকগ্রাউন্ডে (detached mode) চালু করার জন্য। কোনো নতুন কোড চেঞ্জ থাকলে `--build` তা আপডেট করে নেবে। |
| `docker compose up -d` | কোড বিল্ড না করে শুধু এক্সিস্টিং কন্টেইনারগুলো রান করার জন্য। |
| `docker compose down` | চালু থাকা সকল ডকার কন্টেইনার স্টপ এবং রিমুভ করার জন্য। |
| `docker logs -f supremeai` | ব্যাকএন্ড (Spring Boot) কন্টেইনারের লাইভ লগ বা আউটপুট দেখার জন্য। |
| `docker logs -f supremeai-dashboard` | ড্যাশবোর্ড (ফ্রন্টএন্ড) কন্টেইনারের লাইভ লগ দেখার জন্য। |

## ২. লোকাল ডেভেলপমেন্ট: ব্যাকএন্ড (Spring Boot / Java)
ডকার ছাড়া সরাসরি মেশিনে ব্যাকএন্ড চালানোর জন্য।

| কমান্ড | কারণ/ব্যবহার |
| :--- | :--- |
| `./gradlew bootRun` | প্রজেক্টের ব্যাকএন্ড সার্ভার (Spring Boot) লোকালি পোর্ট 8080-এ রান করার জন্য। |
| `./gradlew build` | জাভা প্রজেক্ট কম্পাইল করে `.jar` ফাইল তৈরি করার জন্য (টেস্টসহ)। |
| `./gradlew build -x test` | কোনো ইউনিট টেস্ট রান না করেই প্রজেক্ট বিল্ড করার জন্য (দ্রুত বিল্ডের জন্য)। |
| `./gradlew clean` | আগের বিল্ড করা ফাইল (build ফোল্ডার) মুছে ফেলার জন্য, যাতে নতুন করে ফ্রেশ বিল্ড করা যায়। |

---

## ৩. লোকাল ডেভেলপমেন্ট: ড্যাশবোর্ড (React / Vite)
ড্যাশবোর্ডের ফোল্ডারে (`cd dashboard`) গিয়ে এই কমান্ডগুলো চালাতে হয়।

| কমান্ড | কারণ/ব্যবহার |
| :--- | :--- |
| `npm install` | প্রজেক্টের প্রয়োজনীয় সব প্যাকেজ (node_modules) ইন্সটল করার জন্য। |
| `npm run dev` | ড্যাশবোর্ড লোকাল সার্ভারে (সাধারণত পোর্ট 5173) রান করার জন্য। |
| `npm run build` | প্রোডাকশনে ডেপ্লয় করার জন্য অপটিমাইজড স্ট্যাটিক ফাইল তৈরি করার জন্য। |

## ৪. ফায়ারবেস এবং গুগল ক্লাউড ডেপ্লয়মেন্ট (Firebase & GCP)
সার্ভারে প্রজেক্ট লাইভ করার জন্য এই কমান্ডগুলো প্রয়োজন।

| কমান্ড | কারণ/ব্যবহার |
| :--- | :--- |
| `firebase login` | ফায়ারবেস সিএলআই (CLI) তে আপনার গুগল অ্যাকাউন্ট দিয়ে লগইন করার জন্য। |
| `firebase use supremeai-a` | কাজের জন্য নির্দিষ্ট ফায়ারবেস প্রজেক্ট সিলেক্ট করার জন্য। |
| `firebase deploy --only hosting` | ড্যাশবোর্ডের বিল্ড করা ফাইলগুলো ফায়ারবেস হোস্টিংয়ে লাইভ করার জন্য। |
| `gcloud auth login` | গুগল ক্লাউড (GCP) সিএলআই-তে লগইন করার জন্য। |
| `gcloud config set project supremeai-a` | গুগল ক্লাউড প্রজেক্ট সিলেক্ট করার জন্য। |
| `gcloud run deploy supremeai --source . --region us-central1` | ক্লাউড রানে পুরো ব্যাকএন্ড সোর্স কোড থেকে ডেপ্লয় করার জন্য। |

## ৫. গিট (Git) এবং ভার্সন কন্ট্রোল
কোড সেভ এবং গিটহাবে আপলোড করার জন্য।

| কমান্ড | কারণ/ব্যবহার |
| :--- | :--- |
| `git status` | প্রজেক্টের কোন ফাইলগুলো পরিবর্তন করা হয়েছে তা দেখার জন্য। |
| `git add .` | পরিবর্তিত সকল ফাইল স্টেজিং এরিয়ায় অ্যাড করার জন্য। |
| `git commit -m "Your message"` | পরিবর্তনগুলো নির্দিষ্ট মেসেজ দিয়ে সেভ (কমিট) করার জন্য। |
| `git push origin main` | কমিট করা কোডগুলো গিটহাব রিমোট রিপোজিটরিতে (main ব্রাঞ্চে) পুশ করার জন্য। |
| `git pull` | গিটহাব থেকে লেটেস্ট কোড লোকাল মেশিনে নামিয়ে আনার জন্য। |

## ৬. ট্রাবলশুটিং এবং পোর্ট কিল (Troubleshooting)
অনেক সময় ডকার বা লোকাল সার্ভার রান করতে গেলে "Address already in use" এরর আসে। তখন এগুলো কাজে লাগে।

| কমান্ড | কারণ/ব্যবহার |
| :--- | :--- |
| `fuser -k 8080/tcp` | পোর্ট 8080 (ব্যাকএন্ড) যে প্রসেস ব্যবহার করছে, তা জোরপূর্বক বন্ধ (kill) করার জন্য। |
| `fuser -k 5173/tcp` | পোর্ট 5173 (ড্যাশবোর্ড) যে প্রসেস ব্যবহার করছে, তা জোরপূর্বক বন্ধ করার জন্য। |
| `lsof -i :8080` | পোর্ট 8080-এ কোন অ্যাপ বা প্রসেস চলছে তা দেখার জন্য। |
| `pkill -f bootRun` | ব্যাকগ্রাউন্ডে চলা যেকোনো Spring Boot (bootRun) প্রসেস বন্ধ করার জন্য। |
