# সিস্টেম লার্নিং ট্যাব (System Learning Tab)

## ১. ডিজাইন ও ভিজ্যুয়াল লেআউট

### মূল কম্পোনেন্ট
`dashboard/src/components/SystemLearningDashboard.tsx`

### ভিজ্যুয়াল কম্পোনেন্টস
- **Knowledge Core Stats**: ৪টি গ্লাস কার্ড দিয়ে দেখানো হয় Neural Base Facts, Active Domains, Crawl Velocity, Registry Sync স্ট্যাটাস
- **Knowledge Acquisition Matrix**: ডোমেইনস টেবিল দিয়ে শিখন অগ্রগতি প্রদর্শন
- **Neural Proposals**: রিকমেন্ডেশন কার্ডগুলো দিয়ে নতুন শিখন ক্ষেত্রের প্রস্তাব
- **Stored Intel Snapshot**: সাম্প্রতিক জ্ঞানের তালিকা
- **Live Crawl Stream**: রিয়েলটাইম ক্রল লগের টার্মিনাল

## ২. কীভাবে কাজ করে (Functionality)

### ২.১ মূল ফিচার

#### ডোমেইন রেজিস্টার করা
- `Add Target` বাটনে ক্লিক করে নতুন শিখন ডোমেইন যোগ করা হয়
- ডোমেইন নাম এবং কীওয়ার্ড নির্দিষ্ট করে নতুন শিখন স্তর তৈরি
- এডমিন > API Management > providers-এর সাথে যুক্ত থাকে

#### শিখন প্রক্রিয়া শুরু
- `Activate` বাটনে ক্লিক করে ডোমেইনের জন্য লার্নিং জব ট্রিগার
- WebSocket এর মাধ্যমে রিয়েলটাইম প্রগ্রেস আপডেট
- `ActiveInternetScraper` ব্যবহার করে ওয়েব থেকে তথ্য সংগ্রহ

#### প্রস্তাব অনুমোদন
- Neural Proposals থেকে `Approve` ক্লিক করে নতুন ডোমেইন তৈরি
- `Reject` ক্লিক করে প্রস্তাব বাতিল

### ২.২ API এন্ডপয়েন্ট

| মেথড | এন্ডপয়েন্ট | বর্ণনা |
|------|------------|--------|
| GET | `/api/admin/knowledge/snapshot` | KPI স্ট্যাটাস ডেটা |
| GET | `/api/admin/knowledge/domains` | সকল ডোমেইন তালিকা |
| POST | `/api/admin/knowledge/domains` | নতুন ডোমেইন রেজিস্টার |
| POST | `/api/admin/knowledge/domains/{id}/process` | শিখন প্রক্রিয়া শুরু |
| GET | `/api/admin/knowledge/recommendations` | পেন্ডিং প্রস্তাব |
| POST | `/api/admin/knowledge/recommendations/{id}/approve` | প্রস্তাব অনুমোদন |
| POST | `/api/admin/knowledge/recommendations/{id}/decline` | প্রস্তাব বাতিল |
| GET | `/api/admin/knowledge/recent` | সাম্প্রতিক জ্ঞান এন্ট্রি |

## ৩. ব্যাকএন্ড লজিক (Backend Logic)

```java
@RequestMapping("/api/admin/knowledge")

    // সব endpoint ADMIN role ছাড়া কাজ করে না
}
```

### ৩.২ সার্ভিস লেয়ার (KnowledgeService.java)

#### `registerDomain()`
- নতুন `KnowledgeDomain` অবজেক্ট তৈরি
- Firestore-এ সংরক্ষণ

#### `startLearning()`
- ডোমেইন স্ট্যাটাস `LEARNING`-এ পরিবর্তন
- `ActiveInternetScraper` কল করে ওয়েব স্ক্র্যাপিং

#### `processLearningJob()`
- WebSocket-এ `KNOWLEDGE_CRAWL` ইভেন্ট � Broadcast
- স্ট্রিমিং প্রগ্রেস আপডেট
- `SystemLearning` এন্ট্রি Firestore-এ সংরক্ষণ

#### `getKnowledgeSnapshot()`
- মোট নোড সংখ্যা গণনা
- টপ ৩ ডোমেইন নির্বাচন
- Discovery efficiency হিসাব করে রিটার্ন
