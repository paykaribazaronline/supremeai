# এপিআই ম্যানেজমেন্ট ট্যাব (API Management Tab)

## ১. ডিজাইন ও ভিজ্যুয়াল লেআউট

### মূল কম্পোনেন্ট
`dashboard/src/components/APIManagement.tsx`

### ভিজ্যুয়াল কম্পোনেন্টস
- **Mission Stats**: ৪টি স্ট্যাটাস কার্ড (Active Links, Dead Nodes, API Density, Registry Load)
- **Intelligence Registry**: API প্রোভাইডারদের টেবিল ভিউ
- **Add Provider Modal**: দ্বিপাক্ষীয় লেআউট (বামে Discovery, ডানে Configuration Form)

### ডেটা ইন্টারফেস
```typescript
interface APIProvider {
  id: string;
  name: string;
  type: string;           // llm, image, voice
  apiKey: string;
  status: 'active' | 'inactive' | 'error' | 'limit_exceeded' | 'dead';
  lastTested: string;
  models: string[];
  apiCount?: number;
  usageLimit?: number;
  currentUsage?: number;
}
```

## ২. কীভাবে কাজ করে (Functionality)

### ২.১ মূল ফিচার

#### প্রোভাইডার যোগ করা
- `Establish New Link` বাটনে ক্লিক → Modal ওপেন
- বাম প্যানেলে Internet Discovery: রিয়েলটাইম সার্চ
- ডান প্যানেলে Configuration Form: নাম, টাইপ, API Key, ইমেইল

#### এপিআই কী ভ্যালিডেশন
- `Test Connection` বাটনে ক্লিক
- `/api/admin/providers/test-key` এন্ডপয়েন্টে POST
- রেসপন্সে `valid: true/false` রিটার্ন

#### ইন্টারনেট ডিসকভারি
- ডাউন লেফট করা অক্ষরে Auto-complete ট্রিগার
- `/api/admin/providers/discover?query=` এন্ডপয়েন্ট
- Mock fallback সহ: `${query}-Pro`, `${query}-Vision`

### ২.২ API এন্ডপয়েন্ট

| মেথড | এন্ডপয়েন্ট | বর্ণনা |
|------|------------|--------|
| GET | `/api/admin/providers/configured` | সকল কনফিগার করা প্রোভাইডার |
| POST | `/api/admin/providers` | নতুন প্রোভাইডার যোগ (আপডেটেড) |
| POST | `/api/admin/providers/add` | প্রোভাইডার যোগ (লিগ্যাসি) |
| PUT | `/api/admin/providers/{id}` | প্রোভাইডার আপডেট |
| DELETE | `/api/admin/providers/{id}` | প্রোভাইডার মুছে ফেলা |
| POST | `/api/admin/providers/test-key` | এপিআই কী যাচাই |
| GET | `/api/admin/providers/discover` | মডেল ডিসকভারি |

### ৩.২ API Key যাচাই

```java
@PostMapping("/test-key")
public Mono<ResponseEntity<ApiResponse<Map<String, Object>>>> testProviderKey(
        @RequestBody Map<String, String> payload) {
    String name = payload.get("name");
    String apiKey = payload.get("apiKey");
    
    return discoveryService.validateKey(name, apiKey)
            .map(valid -> valid ? 
                ResponseEntity.ok(ApiResponse.ok(Map.of("valid", true))) :
                ResponseEntity.status(401).body(ApiResponse.error("Invalid key")));
}
```

### ৩.৩ মডেল ডিসকভারি

```java
@GetMapping("/discover")
public Mono<ResponseEntity<ApiResponse<List<Map<String, Object>>>>> discoverModels(
        @RequestParam(required = false) String query) {
    return discoveryService.discoverModels(query)
            .collectList()
            .map(list -> ResponseEntity.ok(ApiResponse.ok(list)));
}
```

### ৩.৪ সেভ ও ডিলিট

```java
@PostMapping
public Mono<ResponseEntity<ApiResponse<Map<String, Object>>>> updateProvider(
        @RequestBody APIProvider provider) {
    String adminUserId = getCurrentAdminUserId();
    return providerRepository.save(provider)
            .flatMap(saved -> {
                ActivityLog log = new ActivityLog();
                log.setUser(adminUserId);
                log.setAction("UPDATE_PROVIDER");
                log.setCategory("PROVIDER_MANAGEMENT");
                // ... log details
                return activityLogRepository.save(log)
                        .thenReturn(ResponseEntity.ok(ApiResponse.ok(...)));
            });
}
```

## ৪. সার্ভিস লেয়ার

### AIProviderDiscoveryService
- `validateKey(name, apiKey)`: এপিআই কী ভ্যালিডেশন
- `discoverModels(query)`: মডেল ডিসকভারি

### ProviderRepository
- Firestore-এ `APIProvider` কালেকশন
- `findAll()`, `save()`, `deleteById()`, `findById()`

## ৫. স্ট্যাটাস ইন্ডিকেশন

| স্ট্যাটাস | রঙ | অর্থ |
|---------|-----|-----|
| active | সবুজ (emerald) | সক্রিয় ও কাজ করছে |
| inactive | নারঙ্গা (orange) | কনফিগার আছে কিন্তু টেস্ট নয় |
| error | লাল (red) | কানেকশন ত্রুটি |
| dead | গাঢ় লাল (red-700) | কী মেয়াদ শেষ বা অবৈধ |
| limit_exceeded | বেগুনি (purple) | ব্যবহার সীমা অতিক্রম |
