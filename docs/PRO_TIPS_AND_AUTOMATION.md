# SupremeAI Pro-Tips & Automation Master Plan

## 1. Conventional Commits

Always use the following prefixes for your commit messages:

- `feat:` New features
- `fix:` Bug fixes
- `style:` Formatting, missing semi-colons, etc.
- `refactor:` Code change that neither fixes a bug nor adds a feature
- `docs:` Documentation changes

## 2. Change Accountability & Rationale

- **The "Why" Rule:** Every commit message and system rule update (`SystemWorkRule.changeNote`) must include a brief explanation of _why_ the change was made, not just _what_ was changed.
- **Format:** Use the commit body (after a blank line) for detailed rationale if the subject line is insufficient.
- **AI Integration:** When using AI-assisted fixes, ensure the AI provides a justification for the logic applied.
