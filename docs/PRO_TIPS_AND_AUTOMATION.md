# SupremeAI Pro-Tips & Automation Master Plan

## 1. Conventional Commits
Always use the following prefixes for your commit messages:
- `feat:` New features
- `fix:` Bug fixes
- `style:` Formatting, missing semi-colons, etc.
- `refactor:` Code change that neither fixes a bug nor adds a feature
- `docs:` Documentation changes

## 2. Infrastructure as Code (IaC)
- Treat your `firebase.json` and `Dockerfile` as critical code. 
- If GCP resources grow, consider moving to **Terraform** for reproducible infrastructure.

## 3. Caching & Performance
- Use GitHub Actions cache for Gradle and NPM to reduce build times by up to 50%.
- In `verify.bat`, always run with `--lint-only` for quick local commits.

## 4. Resilience Patterns
- **Solo-Mode (SK-0054):** Ensure the application always has a `dummy-key` fallback for external AI providers to prevent startup crashes.
- **Self-Healing:** If CI fails, the system automatically sends logs to the Analysis API. Use `verify.bat --auto-heal` to apply generated patches.
