# Git ও CI ফেইলিওরের কারণ ও সঠিক সমাধান

এই ডকুমেন্টটি বর্তমান সমস্যাটি কেন ঘটেছিল এবং ভবিষ্যতে একই ভুল আর না করার জন্য প্রয়োজনীয় নিয়মাবলি ব্যাখ্যা করছে।

## ১. মূল সমস্যার সারসংক্ষেপ

আমাদের CI ছিল ব্যর্থ কারণ গিট/ওয়ার্কফ্লো প্রক্রিয়ায় কয়েকটি ভুল মিলে গেছে:

- `validate_ai.sh` স্ক্রিপ্টটি ভুল API এন্ডপয়েন্টে হিট করছিল (`/api/chat/message`)
- GitHub Actions ওয়ার্কফ্লো লোকাল সার্ভারের ঠিকানা ব্যবহার না করে অনুপযুক্ত ভেরিয়েবল সেট করেছিল
- CI স্টার্টআপ সঠিকভাবে ওজন করা হয়নি; ব্যাকএন্ড হেল্থ চেক না থাকায় ফেইল হলে লগ পাওয়া যেতো না
- `.gitignore`-এ কিছু জেনারেটেড ফাইল ছিল না, ফলে অপ্রত্যাশিত বাইল-ফাইল গিট ট্র্যাকিংয়ে চলে আসছিল
- সিক্রেট/এক্সপ্রেশন ব্যবহারে অদৃশ্য ত্রুটি ছিল, ফলে ওয়ার্কফ্লো ভুল ধাপে আটকে যাচ্ছিল

## ২. কেন এই সমস্যা ঘটেছে

### ২.১ ভুল API এন্ডপয়েন্ট

- প্রকৃত ব্যাকএন্ড এন্ডপয়েন্ট ছিল `/api/chat/send`
- স্ক্রিপ্ট `validate_ai.sh` প্রথমে `/api/chat/message` কল করছিল
- এর ফলে ব্যাকএন্ড HTTP 500 বা 404 রিটার্ন করতে পারে, এবং CI ভ্যালিডেশন ব্যর্থ হয়

### ২.২ লোকাল ব্যাকএন্ডের URL ঠিকমতো ব্যবহার হয়নি

- ওয়ার্কফ্লো ব্যাকএন্ড রান করালেও ভ্যালিডেশন স্ক্রিপ্ট রিমোট বা খালি URL এ পয়েন্ট করতে পারে
- CI-তে `AI_VALIDATION_BASE_URL` সরাসরি `http://127.0.0.1:8080` হওয়া প্রয়োজন, যাতে লোকালি চালিত ব্যাকএন্ডেই পরীক্ষা হয়

### ২.৩ ব্যাকএন্ড স্টার্টআপ চেক ছিল দুর্বল

- ব্যাকএন্ড শুরু হলে তার জন্য পর্যাপ্ত সময় এবং হেল্থ চেক দরকার
- যদি স্বাস্থ্য পরীক্ষা না থাকে, তাহলে ব্যাকএন্ড ভেঙে গেলেও পরে স্ক্রিপ্ট ব্যর্থ হওয়ার আগে স্পষ্ট কারণ পাওয়া যায় না

### ২.৪ জেনারেটেড ফাইল গিটে জমা পড়া

- `public/assets/`, `src/main/resources/static/assets/`, এবং বার বার তৈরি হওয়া build আউটপুট `.gitignore`-এ না থাকলে ওইসব ফাইল ট্র্যাকেড হতে পারে
- এতে কনফ্লিক্ট, বড় diff, ও CI ফাইল সিঙ্ক সমস্যা দেখা দেয়

### ২.৫ GitHub Actions সিক্রেট/ইনপুট ভুল

- ভুল `secrets` বা অনুপযুক্ত এক্সপ্রেশন ব্যবহার করলে ওয়ার্কফ্লো ভুলে ফেলে নো-অপ বা ব্যর্থ স্টেপ চালায়
- CI-তে শুধুমাত্র বাস্তব প্রোডাকশন URL নয়, লোকাল সার্ভার এবং প্রসঙ্গভিত্তিক পরিবেশ ব্যবহার করতে হবে

## ৩. স্থায়ী সমাধান

### ৩.১ এন্ডপয়েন্ট মিলিয়ে নিন

- `scripts/validate_ai.sh`-এ `AI_VALIDATION_ENDPOINT` ডিফল্টে `/api/chat/send` দিন
- ব্যাকএন্ডকে লিগ্যাসি কম্প্যাটিবিলিটি হিসেবে `/api/chat/message` হ্যান্ডেল করতে দিন
- এটি ভবিষ্যতের ইনক্রিমেন্টাল মাইগ্রেশনের জন্য নিরাপদ

### ৩.২ CI-তে লোকাল হোস্ট ঠিক করুন

- `.github/workflows/ai-validation.yml`-এ `AI_VALIDATION_BASE_URL` লোকাল সার্ভার হিসেবে সেট করুন
- `google-github-actions/auth@v2`-এড করলেও সেটিকে ভ্যালিডেশন সার্ভিসের সাথে মিশাবেন না

### ৩.৩ হেল্থ চেক এবং লগিং যোগ করুন

- ব্যাকএন্ড চালানোর পরে `http://localhost:8080/api/health` চেক করুন
- ব্যাকএন্ড না উঠলে `backend.log` দেখিয়ে ফেলুন
- এভাবে CI তাড়াতাড়ি এবং পরিষ্কার কারণে ফেইল হবে

### ৩.৪ `.gitignore` ঠিক করুন

- জেনারেটেড অ্যাসেট, স্ট্যাটিক বিল্ড ফাইল, ও লক বা টেম্প ফাইল `.gitignore`-এ দিন
- যদি আগে ট্র্যাক হয়ে থাকে, `git rm --cached` দিয়ে আনট্র্যাক করুন

### ৩.৫ প্রি-কমিট ও সিকিউরিটি চেক বসান

- `pre-commit` বা `gitleaks` ব্যবহার করে স্থানীয়ভাবে সিক্রেট এবং ফাইল হাইজিন চেক করুন
- নতুন কোড কমিট করার আগে স্টাইল, YAML সিনট্যাক্স, ও সিক্রেট সনাক্তকরণ চলুক

## ৪. একই ভুল আর না করার নিয়মাবলি

### ৪.১ PR রিভিউ চেকলিস্ট

- [ ] CI ওয়ার্কফ্লো কোনো লোকাল রিসোর্সের জন্য সঠিক URL ব্যবহার করছে?
- [ ] API এন্ডপয়েন্ট `send` vs `message` ঠিক আছে?
- [ ] আর্টিফ্যাক্ট বা জেনারেটেড ফাইল গিটে ট্র্যাক করা হচ্ছে না?
- [ ] ব্যাকএন্ড/ফ্রন্টেন্ড স্টার্টআপে হেল্থ চেক ও লগ আছে?
- [ ] সিক্রেট ড্রিফট বা ভুল সিক্রেট ইনজেকশন নেই?

### ৪.২ কমিট বার্তা নিয়ম

- বাগ-ফিক্স করলে উল্লেখ করুন: `fix(ci): validate local backend endpoint and health check`
- ডকুমেন্টেশন করলে রাখুন: `docs: add git CI failure postmortem in Bangla`

### ৪.৩ দলে শেয়ার করা শিক্ষা

- গিট/CI তে `endpoint mismatch` এবং `local URL mismatch` খুব সাধারণ
- এগুলো সার্চ করে দ্রুত ধরা যায়, তাই ব্যাকএন্ড হেল্থ চেক রাখা জরুরি
- কারো কমিট আগে `git status` ও `git diff --cached` দেখে নিন

## ৫. এখনকার স্থিতি

এই সমস্যা এখন সমাধান করা হয়েছে:

- `scripts/validate_ai.sh` এখন সঠিক এন্ডপয়েন্ট ব্যবহার করছে
- `.github/workflows/ai-validation.yml` এখন ইউনিট টেস্ট এবং ব্যাকএন্ড ভ্যালিডেশন আলাদা জবে ভাগ করা হয়েছে, যাতে দ্রুত ফাস্ট টেস্ট আগে চলে এবং ধীর ইন্টিগ্রেশন ভ্যালিডেশন আলাদা থাকে
- AI validation স্ক্রিপ্টে এখন পুনরায় চেষ্টা এবং ব্যাকঅফ সংযোজন আছে, যাতে অস্থায়ী স্টার্টআপ বা নেটওয়ার্ক ফ্লেক ফেইল হয়ে গেলে স্বাভাবিকভাবে পুনরায় পরীক্ষা করে
- ব্যাকএন্ড লগ এবং AI validation report আর্টিফ্যাক্ট হিসেবে সংগ্রহ করা হচ্ছে
- `ChatController.java`-এ লিগ্যাসি `/api/chat/message` রিডাইরেকশন যোগ হয়েছে
- `.gitignore` ও সিকিউরিটি চেক স্ট্রেন্থেন করা হয়েছে
- `ChatControllerTest`-এ JUnit `@Execution(ExecutionMode.SAME_THREAD)` যোগ করা হয়েছে, যাতে টেস্ট স্বয়ংক্রিয়ভাবে প্যারালাল এক্সিকিউশনের কারণে ফ্লেকি না হয়

> ভবিষ্যতে একই ধরনের CI ফেইলিওর এড়াতে এই ডকুমেন্টটি কপি করে PR রিভিউতে ব্যবহার করুন।
